# 내 연구 홈페이지

석사 연구자용 개인 홈페이지입니다. **내 연구 소개 + 읽은 논문 리뷰(.md)** 를 올리는 공간이에요.
화면에 보이는 글자는 영어, 어둡고 모던한 테마이고, 새 논문 리뷰는 **`.md` 파일만 추가하면 자동으로 목록에 올라갑니다.**

> 코딩을 몰라도 됩니다. 아래 순서대로만 따라 하시면 돼요. 막히는 부분은 저(클로드)에게 물어보세요.

---

## 1. 가장 먼저: 내 정보 채우기

`data/profile.json` 파일 **하나만** 고치면 사이트 전체(이름·소개·연락처·링크)가 바뀝니다.
메모장이나 VS Code 같은 편집기로 열어서, **따옴표 `" "` 안의 글자만** 바꾸세요. (화면에 영어로 보이니 영어로 적으시면 됩니다.)

| 항목 | 의미 |
| --- | --- |
| `name` | 이름 |
| `role` | 직함 (예: M.S. Student) |
| `affiliation` | 소속 (학교·대학원·연구실) |
| `field` | 연구 분야 |
| `email` | 공개할 이메일 |
| `tagline` | 첫 화면 한 줄 소개 |
| `about` | 자기소개 문단 |
| `research.summary` | 연구 소개 문단 |
| `research.interests` | 관심 주제 목록 (대괄호 안에 `"항목", "항목"`) |
| `links` | 아래 SNS/링크 목록. **보일 링크는 `url` 채우고, 숨길 링크는 `url`을 `""`(빈칸)로** |

> ⚠️ JSON 파일은 쉼표·따옴표가 빠지면 안 됩니다. 글자만 바꾸고 기호(`{ } , " :`)는 그대로 두세요.

## 2. 새 논문 리뷰 추가하기 (가장 자주 하게 될 일)

1. `reviews/` 폴더에 새 `.md` 파일을 만듭니다.
   - 파일 이름이 곧 주소가 되니 **영문·숫자·하이픈**을 권장해요. 예: `2026-05-paper-name.md`
2. 파일 맨 위에 아래 **머리말**을 붙이고, 그 아래에 자유롭게 본문을 씁니다.

```markdown
---
title: "내가 읽은 논문 제목 리뷰"
date: 2026-05-20
authors: 저자 이름
venue: 학회/저널 이름 (없으면 지워도 됨)
tags: [태그1, 태그2]
link: https://arxiv.org/abs/...   (원문 링크, 없으면 지워도 됨)
summary: 목록 카드에 보일 한 줄 요약
---

여기부터 본문입니다. **굵게**, *기울임*, 목록, 표, 인용 등 마크다운을 자유롭게 쓰세요.
```

3. 올리면(아래 4번) **목록에 자동으로 나타납니다.** (직접 목록을 수정할 필요 없음)

### 본문에서 쓸 수 있는 것

- **수식(LaTeX)**: 줄 안에는 `\( ... \)`, 가운데 정렬은 `$$ ... $$`
  - 예: `$$ E = mc^2 $$`
  - (팁: 인라인 수식에 `_`나 `*`가 많으면 `$ $`보다 `\( \)`가 안전합니다.)
- **코드**: 백틱 3개로 감싸면 색칠됩니다. 언어 이름을 적으면 더 예뻐요.
  - ` ```python ` … ` ``` `
- **이미지**: 그림을 `assets/img/`에 넣고 `![설명](assets/img/그림.png)`
- 표, 인용(`>`), 목록(`-`) 등 일반 마크다운 전부 지원

> 형식이 헷갈리면 `reviews/` 폴더의 예시 파일 2개를 복사해서 내용만 바꾸세요.

## 3. 갤러리에 사진/작업물 추가하기 (선택)

1. 그림을 `assets/img/` 폴더에 넣습니다.
2. `data/gallery.json`의 `items`에 한 칸 추가합니다.

```json
{ "title": "제목", "caption": "설명", "image": "assets/img/내그림.png", "link": "" }
```

## 4. 인터넷에 공개하기 (GitHub Pages — 무료)

처음 한 번만 설정하면, 이후로는 파일만 올리면 자동으로 사이트가 갱신됩니다.

### 처음 1번만 (사이트 만들기)

1. GitHub에 로그인 → 오른쪽 위 **`+` → New repository**.
2. 저장소 이름을 정합니다.
   - **`사용자명.github.io`** 로 만들면 주소가 `https://사용자명.github.io/` 로 깔끔해집니다. (추천)
   - 아무 이름이나 해도 됩니다. 그 경우 주소는 `https://사용자명.github.io/저장소이름/` 이 됩니다. (둘 다 잘 작동)
3. **Create repository** 클릭.
4. 이 폴더(`홈페이지`)의 **모든 파일**을 올립니다. 둘 중 편한 방법:
   - (쉬움) 저장소 화면의 **Add file → Upload files** 에 파일들을 끌어다 놓고 **Commit**.
     - 단, `.github` 폴더와 `.nojekyll` 파일도 꼭 함께 올라가야 합니다. (숨김처럼 보여도 필요)
   - (편함) **GitHub Desktop** 프로그램으로 이 폴더를 올리기.
5. 저장소 상단 **Settings → 왼쪽 메뉴 Pages** 로 이동.
6. **Build and deployment → Source** 를 **`GitHub Actions`** 로 선택.
7. 상단 **Actions** 탭에 들어가면 배포가 진행됩니다. ✅ 초록색 체크가 뜨면 끝.
8. 다시 **Settings → Pages** 로 가면 사이트 주소가 표시됩니다. 접속해 보세요!

### 그 다음부터 (글 추가/수정)

- 새 리뷰 `.md`를 추가하거나 내용을 고친 뒤 **GitHub에 올리기(commit)** 만 하면,
  자동으로 목록이 갱신되고 사이트가 다시 배포됩니다. (몇십 초~1분 정도 걸려요.)
- GitHub 웹사이트에서 바로 **Add file → Create new file** 로 `reviews/제목.md` 를 만들어도 됩니다.

## 5. 내 컴퓨터에서 미리보기 (선택)

> ⚠️ `index.html`을 **그냥 더블클릭하면** 목록·리뷰가 안 보일 수 있습니다.
> (브라우저 보안 때문에 파일을 직접 읽지 못해서예요.) 아래처럼 작은 서버로 열어야 정상 동작합니다.
> 번거로우면 그냥 4번으로 공개한 뒤 실제 주소에서 확인해도 됩니다.

가장 쉬운 방법 — **`미리보기.bat` 더블클릭**:
1. 이 폴더의 **`미리보기.bat`** 파일을 더블클릭합니다.
2. 검은 창이 뜨고, 잠시 후 브라우저가 `http://localhost:8000` 으로 자동으로 열립니다.
3. 다 보고 나면 **검은 창을 닫으면** 종료됩니다.
   - (이미 설치된 Python을 사용합니다. 빨간 글씨로 `python ... 찾을 수 없습니다` 오류가 뜨면 알려주세요.)

또는 **VS Code + Live Server**:
1. [VS Code](https://code.visualstudio.com/) 설치 → 이 폴더 열기.
2. 확장(Extensions)에서 **Live Server** 설치.
3. `index.html`에서 마우스 오른쪽 → **Open with Live Server**.

## 폴더 설명 (참고용)

```
홈페이지/
├── index.html / reviews.html / review.html / gallery.html  ← 화면들
├── assets/         ← 디자인(css), 기능(js), 이미지(img)
├── data/
│   ├── profile.json   ← ★ 내 정보 (여기 수정)
│   └── gallery.json   ← 갤러리 항목
├── reviews/
│   ├── *.md           ← ★ 논문 리뷰 (여기에 추가)
│   └── manifest.json  ← 리뷰 목록(자동 생성, 직접 수정 X)
├── scripts/build.mjs  ← 목록 자동 생성기
├── 미리보기.bat        ← 로컬 미리보기 실행 파일
└── .github/workflows/deploy.yml  ← 자동 배포 설정
```

## 자주 묻는 것

- **리뷰를 지우려면?** `reviews/`에서 그 `.md` 파일을 삭제하고 올리면 목록에서도 사라집니다.
- **`manifest.json`을 직접 고쳐야 하나요?** 아니요. 올릴 때 자동으로 다시 만들어집니다.
- **디자인 색을 바꾸려면?** `assets/css/style.css` 맨 위 `:root`의 색상 값만 바꾸세요.
